# special-parakeet

Student Project – Scope of Work (SOW)
Project Title: OpenClaw Memory & Policy Console
Student Name: Justin Chen
Date: 4/6/2026

1. Project Overview
I am to create an application that stores structured memories learned from chats, manages platform-specific contacts and permissions, and saves all data using long-term file storage. The program will support both normal user interaction through menus and a strict command mode for safe structured input from an external AI system such as OpenClaw and NemoClaw.


2. Features
Example: Ability to add new usersAbility to add new contacts
Ability to store contacts by platform, such as WhatsApp or Telegram
Ability to assign contact security fields such as privilege tier, active/inactive status, and verification status
Ability to add new memory records learned from chats
Ability to store memory details such as time, person, platform, summary, importance, and security level
Ability to use strict command mode with structured inputs (e.g  ADD_MEMORY|... and ADD_CONTACT|... ) 
Ability to search memories by time/person
Ability to import structured md files and store their contents into program
Ability to store command policies for access control
Ability to check whether a user/contact is allowed to perform a command, also to separate user actions from agent actions.
Ability to keep audit logs of actions and denied/accepted operations


3. Deliverables
Program files
Persistent storage files such as contacts.csv, core_mem.csv, commands.csv, audit_log.txt…
Output.pdf with screenshots and explanation
GitHub submission with correct file structure
Final PowerPoint / slideshow as PDF
YouTube video presentation with program explanation and demo

4. Timeline
Week 1: Define idea, set up GitHub, design file formats and data structures
Week 2: Build contact registry and long-term storage
Week 3: Build memory registry and search by time/person
Week 4: Add markdown import and command policy storage
Week 5: Add strict command mode, audit logging, testing, and debugging
Week 6: Final documentation, screenshots, PowerPoint/PDF, YouTube demo, and submission



5. Success Criteria
Program runs correctly, saves data with long-term file storage, includes required features, and is submitted with code, documentation, screenshots, presentation, and demo video on time. Also includes a simple test comparing how accurately different models format and use the program’s strict command inputs.
